DATA PRIVACY & SECURITY STATEMENT: MADE ACOUSTIC EXTRACTION PIPELINE
===============================================

1. Local-First Processing Architecture
--------------------------------------------------------------------------------
MADE is designed with a "local-first" architecture to ensure the security of 
sensitive research data. Unlike cloud-based acoustic analysis services, all 
feature extraction is performed on the researcher's local hardware.

Acoustic Analysis: The pipeline utilizes Praat via the parselmouth Python 
bindings for all signal processing and feature extraction. All computations 
occur locally using your system's CPU.

Audio/TextGrid I/O: File reading and writing operations using praatio and 
pandas are conducted entirely within your local environment. No data is 
cached or transmitted to external servers.

Feature Extraction: All 26 feature categories (pitch, formants, voice quality, 
MFCC, etc.) are calculated locally. The modular extractor architecture ensures 
that raw audio is accessed only during the extraction process and not retained 
after each file is processed.

2. Management of Personally Identifiable Information (PII)
--------------------------------------------------------------------------------
Researchers should be aware that certain files processed and generated by 
MADE contain biometric data and linguistic content that may qualify as PII:

Raw Audio Files (.wav): These contain biometric voice data that can uniquely 
identify individual speakers. MADE reads these files but does not modify them.

TextGrid Files: These contain time-aligned transcriptions that may include 
participant names, locations, or other identifiable information extracted 
from the audio content.

Extracted Features (.csv): While aggregated acoustic features (e.g., mean pitch, 
formant values) are less obviously identifiable, research has shown that 
sufficient feature combinations can potentially re-identify speakers. These 
output files should be treated as sensitive data.

Speaker Identification: MADE does not perform explicit speaker diarization 
or fingerprinting (unlike the ASR pipeline). However, extracted features 
such as pitch range, voice quality (jitter/shimmer), and formant values can 
vary systematically across speakers and may serve as de facto biometric 
identifiers when combined.

Storage Warning: All output files (.csv) and any intermediate data should be 
handled, stored, and shared with the same level of security as the original 
raw audio files to comply with IRB and data protection protocols (e.g., GDPR, 
HIPAA, FERPA).

3. External Connectivity and Authentication
--------------------------------------------------------------------------------
MADE is designed for entirely offline operation after initial installation:

No Authentication Required: Unlike the ASR pipeline, MADE does not require 
Hugging Face tokens, API keys, or any form of authentication.

No Model Downloads: All acoustic analysis uses the local Praat installation 
accessed via parselmouth. No pre-trained models are downloaded from external 
repositories during execution.

No Telemetry: MADE contains no telemetry, usage tracking, crash reporting, 
or automatic update checking. The only external connectivity occurs when 
users manually install dependencies via pip or conda.

Zero Data Transmission: During execution, no audio data, TextGrid content, 
or extracted features are ever transmitted to external servers or third parties.

4. Data Lifecycle and Intermediate Files
--------------------------------------------------------------------------------
MADE generates minimal intermediate files that researchers must manage:

Extracted Features (.csv): The primary output file containing all requested 
acoustic measures. This file is written once at the completion of extraction.

Temporary Objects: MADE creates temporary Praat objects (Pitch, Formant, 
Intensity, MFCC objects) during extraction. These exist only in memory and 
are destroyed after each interval is processed. No temporary files are 
written to disk.

Logs: MADE does not create persistent log files by default. Progress 
information is printed to the terminal and not saved unless explicitly 
redirected by the user.

Directory Structure: MADE creates a ./libs/ directory for extractor modules 
but writes no user data there. The pipeline does not create hidden cache 
directories or store processing artifacts.

Data Retention Policy: Raw audio and TextGrid files are opened, read, and 
closed for each interval. MADE does not retain file handles, memory buffers, 
or cached representations of audio data after processing each file.

5. Researcher Responsibility
--------------------------------------------------------------------------------
This software is a tool for local acoustic feature extraction. It is the 
responsibility of the researcher to ensure that:

Hardware Security: The computer used for processing is physically secure 
and protected from unauthorized access.

Access Controls: Output files (.csv) are stored with appropriate file 
permissions and encryption where required by institutional policy.

Data Minimization: Only necessary features are extracted. The interactive 
selection system allows researchers to limit extraction to only the 
acoustic measures required for their analysis.

Retention Policies: Extracted feature files are retained only as long as 
necessary for the research purpose, in accordance with approved data 
management plans.

Institutional Compliance: All processing and storage complies with their 
institution's data management plan, IRB protocols, and applicable privacy 
laws (e.g., GDPR, HIPAA, FERPA, PIPEDA).

Participant Protections: Any data shared with collaborators (including 
aggregated feature tables) is properly de-identified and shared only through 
secure channels approved by their institution.

6. Comparison with Cloud-Based Alternatives
--------------------------------------------------------------------------------
| Aspect | MADE (Local) | Cloud ASR/Acoustic APIs |
|--------|--------------|--------------------------|
| Audio transmission | None | Uploaded to vendor servers |
| Data retention | User-controlled | Vendor-dependent policies |
| Third-party access | None | Possible for training/improvement |
| Network requirement | Only for install | Required for every file |
| Compliance | Self-certified | Vendor-dependent (may violate IRB) |
| PII exposure | Local only | Potential cloud breach risk |

MADE provides complete data sovereignty, ensuring that sensitive biometric 
voice data and associated transcriptions never leave the researcher's 
controlled environment.

7. Security Recommendations
--------------------------------------------------------------------------------
For enhanced security when using MADE with sensitive data:

- Run MADE on an air-gapped or institutionally managed computer
- Encrypt your raw audio and TextGrid storage using LUKS, BitLocker, or VeraCrypt
- Store output CSV files on encrypted volumes
- Use secure deletion tools (e.g., srm, shred) for temporary copies
- Implement access logging for directories containing audio and extracted features
- Consider differential privacy techniques for shared feature tables
- Anonymize TextGrid transcriptions by removing participant names before processing

8. Vulnerability Reporting
--------------------------------------------------------------------------------
If you discover a security vulnerability in MADE that could lead to data 
exposure, please report it responsibly by:

1. Emailing the maintainer directly (address in CITATION file)
2. Not disclosing the vulnerability publicly until a patch is available
3. Providing sufficient detail to reproduce and address the issue

MADE is provided "as is" without warranty. Users are responsible for 
validating that the tool meets their security requirements.
